# Universal Content Architecture Decision Tree Knowledge Document

## Decision Tree: Selecting Content Type

```
Start: What is the primary outcome needed?
│
├─> Need Understanding? → EXPLANATORY
│   └─> "Audience needs to understand what something is and why it matters"
│
├─> Need Decision? → DECISION-SUPPORT
│   └─> "Audience needs to choose between options or evaluate solutions"
│
├─> Need Action? → PROCESS-ORIENTED
│   └─> "Audience needs to complete specific tasks or implement procedures"
│
└─> Need Awareness? → AWARENESS-BUILDING
    └─> "Audience needs to know about trends, changes, or opportunities"
```

---

## EXPLANATORY Content Structure

### Purpose
Transform complex concepts into clear understanding, enabling audience to grasp fundamentals and explain to others.

### Opening (100-150 words)
- **Format:** Problem scenario or knowledge gap
- **Elements:** Relatable situation, pain point, why it matters now
- **Hook:** Start with a question or challenge they face
- **Example:** "You're in a meeting when someone mentions [technical concept] and everyone nods knowingly. You nod too, but inside you're wondering what it actually means and why it matters to your work..."

### Core Concept (200-300 words)
- **Format:** Definition + significance + impact
- **Structure:** 
  1. What it is (simple definition)
  2. Why it exists (problem it solves)
  3. How it helps (benefits/value)
- **Elements:** Clear definition, business relevance, practical impact
- **Progression:** Simple → Detailed → Applied

### Practical Application (300-400 words)
- **Format:** 3 specific examples with increasing complexity
- **Structure per example:**
  - Scenario setup (2-3 sentences)
  - How concept applies (3-4 sentences)
  - Outcome/benefit (2-3 sentences)
- **Progression:** 
  1. Simple/Individual level
  2. Team/Department level
  3. Organization/Strategic level

### Implementation Reality (150-200 words)
- **Format:** Honest assessment of requirements
- **Elements to address:**
  - Time investment needed
  - Budget/resource requirements
  - Skill prerequisites
  - Common challenges
  - Realistic timeline
- **Tone:** Balanced optimism with transparency

### Next Steps (100-150 words)
- **Format:** Immediate action + learning pathway
- **Components:**
  - One 30-minute task to start today
  - Resources for deeper learning
  - Progression to next level
  - Success indicators
  - Support resources

### Content Length by Level
| Level 1 | Level 2 | Level 3 |
|---------|---------|---------|
| 600-800 words | 800-1,200 words | 1,200-1,600 words |

## DECISION-SUPPORT Content Structure

### Purpose
Enable informed decision-making by providing frameworks, criteria, and comparative analysis.

### Opening (100-150 words)
- **Format:** Decision scenario with stakes
- **Elements:** Choice to make, pressure points, consequences
- **Hook:** Present the dilemma they're facing
- **Example:** "You need to choose a [solution type] for your team. The wrong choice could cost months of rework and thousands in wasted budget. The right choice could transform your operations..."

### Decision Framework (250-350 words)
- **Format:** Structured evaluation criteria
- **Components:**
  1. Critical factors (must-haves)
  2. Important factors (should-haves)
  3. Nice-to-have factors
  4. Weighting methodology
  5. Red flags to avoid
- **Tools:** Scoring matrix, decision tree, or rubric

### Option Analysis (400-600 words)
- **Format:** 2-4 viable options with detailed comparison
- **Per option structure:**
  - Overview (2-3 sentences)
  - Strengths (3-4 points)
  - Limitations (3-4 points)
  - Best fit scenarios (2-3 examples)
  - Cost/resource implications
- **Comparison format:** Side-by-side or sequential

### Resource Requirements (200-300 words)
- **Format:** Detailed resource analysis per option
- **Elements per option:**
  - Initial investment (time, money, people)
  - Ongoing costs
  - Skill requirements
  - Infrastructure needs
  - Hidden costs/considerations
- **Presentation:** Table or structured list

### Decision Tools (150-250 words)
- **Format:** Practical decision-making instruments
- **Components:**
  - Scoring rubric (downloadable/copyable)
  - Evaluation checklist
  - ROI calculator framework
  - Risk assessment matrix
- **Usage:** Step-by-step instructions for tools

### Content Length by Level
| Level 1 | Level 2 | Level 3 |
|---------|---------|---------|
| 800-1,200 words | 1,200-1,500 words | 1,500-2,000 words |

## PROCESS-ORIENTED Content Structure

### Purpose
Enable successful execution of specific procedures, implementations, or workflows.

### Opening (150-200 words)
- **Format:** Process overview with end goal
- **Elements:** What you'll accomplish, why it matters, success criteria
- **Hook:** Paint picture of successful outcome
- **Example:** "By the end of this guide, you'll have successfully [specific outcome]. This process typically takes [timeframe] and will enable you to [key benefit]..."

### Preparation (200-300 words)
- **Format:** Prerequisites and setup requirements
- **Sections:**
  1. Knowledge prerequisites
  2. Tool/system requirements
  3. Access/permissions needed
  4. Data/information to gather
  5. Stakeholder notifications
- **Checklist:** Pre-flight verification items

### Step-by-Step Instructions (600-1,000 words)
- **Format:** Numbered steps with sub-steps
- **Per step structure:**
  - Action headline (starts with verb)
  - Detailed instructions
  - Expected outcome
  - Common variations
  - Decision points (if applicable)
  - Validation checkpoint
- **Supporting elements:** Screenshots, diagrams, examples

### Troubleshooting (200-400 words)
- **Format:** Problem-solution pairs
- **Structure per issue:**
  - Symptom description
  - Diagnostic questions
  - Root cause
  - Solution steps
  - Prevention tips
- **Organization:** Most common to least common

### Validation (150-250 words)
- **Format:** Success verification and next actions
- **Components:**
  - Success indicators checklist
  - Quality metrics
  - Common mistakes to verify against
  - Next steps after completion
  - Ongoing maintenance needs

### Content Length by Level
| Level 1 | Level 2 | Level 3 |
|---------|---------|---------|
| 1,000-1,500 words | 1,200-2,000 words | 2,000-2,500 words |

## AWARENESS-BUILDING Content Structure

### Purpose
Create awareness of trends, changes, or opportunities that require attention or preparation.

### Opening (100-150 words)
- **Format:** Trend context and urgency
- **Elements:** What's happening, velocity of change, relevance
- **Hook:** Why they can't afford to ignore this
- **Example:** "While you've been focused on daily operations, [trend/change] has been quietly reshaping your industry. Organizations that don't adapt within [timeframe] risk..."

### Impact Analysis (200-300 words)
- **Format:** How this affects their world
- **Sections:**
  1. Immediate impacts (0-6 months)
  2. Medium-term impacts (6-18 months)
  3. Long-term impacts (18+ months)
- **Per impact:** Specific changes to work, processes, or markets
- **Tone:** Balanced (opportunities + challenges)

### Opportunity Assessment (150-250 words)
- **Format:** What this enables or threatens
- **Components:**
  - New capabilities unlocked
  - Competitive advantages available
  - Risks of inaction
  - First-mover advantages
  - Market positioning implications

### Preparation Guidance (100-200 words)
- **Format:** How to get ready
- **Elements:**
  - Skills to develop
  - Knowledge to acquire
  - Resources to secure
  - Partnerships to build
  - Experiments to run

### Monitoring Framework (50-100 words)
- **Format:** How to stay current
- **Components:**
  - Key sources to follow
  - Signals to watch for
  - Update frequency recommended
  - Decision triggers
  - Review cadence

### Content Length by Level
| Level 1 | Level 2 | Level 3 |
|---------|---------|---------|
| 400-600 words | 600-1,000 words | 1,000-1,400 words |

## Architecture Selection Matrix

| If the audience needs to... | Choose... | Because... |
|----------------------------|-----------|------------|
| Understand a new concept | Explanatory | Builds foundational knowledge |
| Choose between options | Decision-Support | Provides evaluation framework |
| Complete a specific task | Process-Oriented | Offers step-by-step guidance |
| Know about emerging trends | Awareness-Building | Creates strategic awareness |

---

## Content Flow Patterns

### Explanatory Flow
```
Confusion → Context → Concept → Application → Action
```

### Decision-Support Flow
```
Dilemma → Criteria → Options → Analysis → Choice
```

### Process-Oriented Flow
```
Goal → Preparation → Execution → Validation → Optimization
```

### Awareness-Building Flow
```
Signal → Impact → Opportunity → Preparation → Monitoring
```

---

## Section Balance Guidelines

### Explanatory Balance
- 20% Opening and context
- 30% Core concept explanation
- 35% Practical applications
- 15% Next steps and resources

### Decision-Support Balance
- 15% Opening scenario
- 25% Framework development
- 40% Option analysis
- 20% Tools and resources

### Process-Oriented Balance
- 15% Overview and prep
- 60% Step-by-step instructions
- 15% Troubleshooting
- 10% Validation and next steps

### Awareness-Building Balance
- 20% Trend context
- 35% Impact analysis
- 25% Opportunities
- 20% Preparation and monitoring

## Quality Checkpoints by Architecture

### Explanatory Checkpoints
- [ ] Concept clearly defined within first 200 words
- [ ] 3+ progressive examples included
- [ ] Connection to daily work established
- [ ] Learning pathway identified

### Decision-Support Checkpoints
- [ ] All options fairly presented
- [ ] Evaluation criteria objective
- [ ] Resource requirements transparent
- [ ] Decision tool provided

### Process-Oriented Checkpoints
- [ ] Every step has validation
- [ ] Common errors addressed
- [ ] Prerequisites clearly stated
- [ ] Success metrics defined

### Awareness-Building Checkpoints
- [ ] Urgency established
- [ ] Impacts clearly articulated
- [ ] Opportunities identified
- [ ] Monitoring plan provided
